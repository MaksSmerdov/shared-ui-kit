# Версии `@sorbent/ui-kit`

Этот файл фиксирует соответствие между версиями пакета и волнами миграции из `new-main-server`.

## Принцип версионирования на текущем этапе

Пока пакет развивается волнами переноса, каждая завершенная волна считается отдельной версией пакета:

- `0.1.0` — первая волна
- `0.2.0` — вторая волна
- `0.3.0` — третья волна

Это не финальная release policy, а текущий рабочий договор для этапа активной миграции.

## `0.1.0`

Первая волна переноса.

### Что вошло

- `Button`
- `Tooltip`
- `IconButton`
- `Loader`
- `Skeleton`
- `UiKitProvider`
- базовый light/dark theme layer

### Что это означало

- появился первый публичный API пакета;
- появился минимальный theme contract;
- пакет стал самостоятельным базовым UI-слоем поверх `MUI`.

## `0.2.0`

Вторая волна переноса.

### Что вошло

- `Input`
- `Checkbox`
- `DatePicker`
- `Select`
- `Accordion`
- `inputClassNames`

### Что это означало

- в пакет переехал основной набор form-controls;
- `new-main-server/frontend` был переведен на пакетные импорты второй волны;
- локальные legacy-реализации второй волны были удалены из consumer-проекта.

## `0.3.0`

Третья волна переноса.

### Что вошло

- `Modal`
- `SearchInput`
- `ErrorMessage`
- `DropdownMenu`
- `FullscreenOverlay`

### Что это означало

- пакет расширился от базовых контролов до reusable UI-patterns;
- `new-main-server/frontend` был переведен на пакетные импорты третьей волны;
- локальные legacy-реализации третьей волны были удалены из consumer-проекта;
- текущая версия пакета на этом этапе — `0.3.0`.

## Что пока не вынесено в версионный контракт

На данный момент в отдельные версии еще не оформлены:

- `Snackbar`
- `Sidebar`

Для них нужен отдельный аудит, потому что они сильнее связаны с приложением и его runtime-контекстом.

## Как использовать этот файл

Этот файл нужен, чтобы быстро ответить на вопросы:

- какая версия соответствует какой волне;
- что именно вошло в конкретный релиз;
- с какого релиза тот или иной компонент считается частью публичного API.
